> This file is loaded by the edit-review agent before every editing session.

## 1. Three-Pass Editing Methodology

Professional editing is executed in three sequential passes. Each pass has a
single focus — never mix concerns across passes.

### Pass 1 — Structural Edit (Developmental)

Look at the piece as a whole. Ignore grammar. Focus on architecture.

- [ ] **Thesis / core argument** — Is it stated clearly within the first 20% of the piece?
- [ ] **Hook** — Does the opening create genuine curiosity or urgency? Would you keep reading?
- [ ] **Logical flow** — Does each section follow naturally from the previous one?
- [ ] **Section weight** — Are sections proportional to their importance? Flag any that drag.
- [ ] **Redundancy** — Are there repeated ideas, even if worded differently? Consolidate.
- [ ] **Transitions** — Does every section link to the next with a clear bridge sentence?
- [ ] **Evidence / support** — Are claims backed by data, examples, or stories? Flag unsupported assertions.
- [ ] **CTA clarity** — Is there a single, unambiguous call-to-action? Does it match the piece's intent?
- [ ] **Audience alignment** — Is the piece pitched at the right level for the target reader?
- [ ] **Format fit** — Does the structure match the medium (blog, newsletter, social, whitepaper)?
- [ ] **Headline / title** — Does it promise what the piece delivers? Is it specific enough?
- [ ] **Subheadings** — Can a reader skim subheadings alone and understand the argument?
- [ ] **Conclusion** — Does it land with impact, or does it trail off?

### Pass 2 — Line Edit (Copyedit)

Now zoom in. Work sentence by sentence.

- [ ] **Sentence length** — Vary it. Break any sentence over 25 words unless rhythm demands it.
- [ ] **Paragraph length** — Max 3-4 sentences for digital content. One-sentence paragraphs are fine for emphasis.
- [ ] **Filler words** — Remove: just, really, very, quite, actually, basically, literally, in order to.
- [ ] **Weak verbs** — Replace "is/was/are" constructions with active verbs where possible.
- [ ] **Passive voice** — Convert to active unless passive is intentionally used for emphasis or diplomacy.
- [ ] **Adverb pruning** — If the verb is strong enough, the adverb is redundant. "He sprinted" not "He ran quickly."
- [ ] **Cliches** — Flag and replace: "at the end of the day," "game-changer," "move the needle," "deep dive."
- [ ] **Jargon check** — Is every technical term either defined or universally understood by the audience?
- [ ] **Pronoun clarity** — Every "it," "this," "they" must have an unambiguous antecedent.
- [ ] **Grammar and punctuation** — Subject-verb agreement, comma splices, dangling modifiers, apostrophe misuse.
- [ ] **Parallel structure** — Lists and comparisons must use consistent grammatical form.
- [ ] **Tense consistency** — No unexplained shifts between past, present, and future.
- [ ] **Word repetition** — Flag the same word used more than twice in a paragraph (excluding articles/prepositions).
- [ ] **Contractions** — Use them for conversational tone; avoid for formal/authoritative pieces.
- [ ] **Numbers** — Spell out one through nine; use numerals for 10+. Be consistent.
- [ ] **Oxford comma** — Follow the style guide. Be consistent throughout the piece.
- [ ] **Link text** — Never use "click here" or "read more." Use descriptive anchor text.

### Pass 3 — Brand Voice Check

Compare the piece against the brand DNA and style guide.

- [ ] **Voice characteristics** — Does the piece embody each stated brand voice trait?
- [ ] **Banned words** — Are any words from the "never use" list present?
- [ ] **Preferred terms** — Are brand-preferred terms used consistently?
- [ ] **Tone match** — Does the tone fit the content type? (e.g., newsletters can be warmer; thought leadership should be more authoritative)
- [ ] **Personality test** — Read aloud. Does it sound like the brand, or like a generic content mill?
- [ ] **Value alignment** — Does the piece reinforce or contradict brand values?
- [ ] **Competitor language** — Are we accidentally using a competitor's signature phrases?
- [ ] **Consistency** — Is the voice consistent from start to finish, or does it shift mid-piece?
- [ ] **"We" vs "I" vs "You"** — Is the POV correct for this content type per the style guide?
- [ ] **Emotional register** — Does the emotional intensity match the topic and platform?

## 2. Common Writing Mistakes and Fixes

### Structural Mistakes

| Mistake | Fix | Example |
|---------|-----|---------|
| **Buried lede** | Move the most compelling point to the opening. The lede is often hiding in the last paragraph. | Before: 3 paragraphs of background, then the insight. After: Lead with the insight, backfill context. |
| **Committee voice** | Rewrite sections that shift tone mid-piece. One voice throughout. | "We leverage synergies" followed by "Here's the real talk" — pick one register. |
| **Missing CTA** | Every piece needs a next step. Add one that matches the content's intent. | Blog post about a problem ends without pointing to a solution or resource. |
| **Wall of text** | Break into scannable sections with subheadings, bullet points, and white space. | A 500-word unbroken paragraph becomes 3 short paragraphs + a bulleted list. |

### Sentence-Level Mistakes

| Mistake | Fix | Example |
|---------|-----|---------|
| **Run-on sentences** | Split at the conjunction or use a semicolon. | Before: "She went to the store and she forgot the milk and then she had to go back." After: "She went to the store but forgot the milk. She had to go back." |
| **Comma splice** | Replace comma with semicolon, period, or add a conjunction. | Before: "The project failed, nobody was surprised." After: "The project failed; nobody was surprised." |
| **Dangling modifier** | Attach the modifier to the correct subject. | Before: "Running through the park, the rain started." After: "Running through the park, she felt the rain start." |
| **Nominalizations** | Convert noun forms back into verbs for energy. | Before: "The implementation of the strategy..." After: "We implemented the strategy..." |
| **Redundant phrases** | Cut to the essential word. | "Absolutely essential" -> "essential." "Past history" -> "history." "End result" -> "result." |
| **Weasel words** | Replace vague qualifiers with specifics or remove them. | Before: "Some experts say this might possibly help." After: "Three studies show this works." |
| **Flabby claims** | Back every claim with evidence or delete it. | Before: "We're the industry leader." After: "We've served 12,000 clients across 40 countries." |

### Voice and Tone Mistakes

| Mistake | Fix | Example |
|---------|-----|---------|
| **Tone-deaf opening** | Match energy to audience and context. | An upbeat intro on a sensitive topic — rewrite to acknowledge the gravity first. |
| **Over-formality** | Use contractions, shorter sentences, direct address. | Before: "One might consider the implications." After: "Think about what this means for you." |
| **Under-formality** | Remove slang, add specificity, strengthen evidence. | Before: "This hack is totally fire." After: "This technique consistently outperforms the standard approach." |

## 3. Readability Scoring — Flesch-Kincaid Targets

### Flesch Reading Ease (FRE) — Target Ranges by Content Type

| Content Type | FRE Target | Grade Level | Why |
|-------------|-----------|-------------|-----|
| **Social media posts** | 70–80 | 6–7 | Skimmed on mobile. Must be instantly digestible. |
| **Blog posts** | 60–70 | 7–9 | Accessible to 85%+ of adults. Allows some depth. |
| **Newsletter** | 65–75 | 7–8 | Conversational but substantive. Readers skim quickly. |
| **Landing pages** | 70–80 | 6–7 | Conversion-focused. Clarity drives action. |
| **Thought leadership** | 50–60 | 10–12 | Audience expects depth. Complexity is acceptable. |
| **Technical docs** | 40–50 | 12–14 | Expert audience. Precision over accessibility. |
| **Email outreach** | 65–75 | 7–8 | Must respect the reader's time. Skimmable. |
| **Case studies** | 55–65 | 9–11 | Balances narrative with data. Professional audience. |

### How to Estimate Without a Calculator

- **Average sentence length:** Count words in 5 random sentences, average them. Target: 15–20 words.
- **Syllable density:** If more than 20% of words have 3+ syllables, simplify.
- **Quick test:** If you need to re-read a sentence to understand it, it is too complex.

### When a Piece Misses the Target

- FRE too low (too hard): Break long sentences. Replace multisyllabic words. Add subheadings.
- FRE too high (too simple): Combine short choppy sentences. Add nuance. Appropriate for the audience?
- Off by 5+ points: Flag in editor's note. Suggest specific sentences to revise.

## 4. Brand Voice Checklist

Run this checklist on every piece after the line edit is complete. Load the
brand DNA (`data/brand/brand-dna.md`) and style guide (`data/brand/style-guide.md`)
before starting.

### Pre-Check: Load Brand Assets
- [ ] Read brand DNA — note the core voice traits (e.g., "bold, direct, human")
- [ ] Read style guide — note preferred/banned words, formatting rules, POV conventions
- [ ] Identify the content type and match to the appropriate tone variant

### Voice Trait Verification
For each stated brand voice trait, answer:
- [ ] Can I point to at least 3 sentences that demonstrate this trait?
- [ ] Are there any sentences that contradict this trait?
- [ ] If I removed the byline, would the reader still recognize this as our brand?

### Lexicon Compliance
- [ ] No words from the "never use" list appear in the piece
- [ ] Brand-preferred terms are used (not synonyms or alternatives)
- [ ] Product names, service names, and brand terms are capitalized/formatted correctly
- [ ] Industry jargon is either defined or excluded per style guide rules

### Tone Calibration
- [ ] Tone matches the platform (LinkedIn vs. newsletter vs. blog)
- [ ] Emotional register is appropriate (not too hot, not too cold)
- [ ] Humor, if present, aligns with brand personality
- [ ] Formality level matches the style guide's directive for this content type

### Consistency Sweep
- [ ] Voice does not shift between sections (common when pieces are drafted over multiple sessions)
- [ ] POV (we/I/you) is consistent throughout
- [ ] The opening and closing feel like they were written by the same person

## 5. Editor's Note Format Standards

Every edited piece must include an editor's note. This is the editor's
professional accountability artifact.

### Structure

1. **Changes Summary** — 2-4 bullet points describing the major edits made. Group by type (structural, line, voice).
2. **Flags** — Questions, concerns, or ambiguities for the author or the CEO agent/user to resolve. These are things the editor cannot fix unilaterally.
3. **Verdict** — One of three ratings:
   - **Pass** — Ready for publication with no further edits.
   - **Needs Revision** — Specific issues must be addressed before publication. List them.
   - **Reject** — Piece requires a fundamental rework. Explain why (see Section 6).
4. **Edit Log** — A detailed list of specific changes with brief rationale. Format: "Changed X to Y — reason."

### Tone

- Professional, not harsh. The goal is to help the author improve.
- Be specific. "Tightened the intro" is useless. "Cut 40 words from intro; moved the market-size stat to sentence 1 for a stronger hook" is useful.
- Flag subjective changes clearly: "Suggested change — author may prefer original."

### Length

- Changes Summary: 50-100 words
- Flags: As many as needed, but each should be 1-2 sentences
- Edit Log: No limit, but be concise per item

## 6. When to Suggest a Rewrite vs. an Edit

This is the most consequential decision the editor makes. Use this framework.

### Suggest an EDIT when:

- The core argument is sound but the execution needs polish
- The structure works but individual sections need tightening
- Voice is mostly on-brand with a few off-notes
- Readability score is within 10 points of the target
- The piece would be publication-ready after the three-pass process
- Estimated editing effort: under 40% of the text needs changes

### Suggest a REWRITE when:

- The piece fundamentally misses the target audience
- The core argument is flawed, unclear, or unsupported
- Structure is disorganized beyond section rearrangement
- Voice is consistently off-brand (reads like a different company wrote it)
- Readability score is 15+ points from target
- More than 60% of sentences need revision
- The hook, body, and CTA are all misaligned

### The Summary Test

Write a one-sentence summary of what the draft says, then write a one-sentence
summary of what it should say. If they describe the same idea with different
execution, edit. If they describe different ideas, rewrite.

### How to Communicate a Rewrite Recommendation

- Never say "this is bad." Say "this draft has potential but needs a structural rework to hit the mark."
- Identify 1-2 things that ARE working, so the author knows what to preserve.
- Provide a clear brief for the rewrite: target audience, core argument, desired structure, tone.
- Set the verdict to "Reject" in the editor's note, with the rewrite brief attached.

## 7. Additional Guidelines

### Reading Aloud

Every piece should pass the "read aloud" test. If a sentence is awkward to
speak, it is awkward to read. This catches:
- Unnatural rhythm
- Tongue-twister phrasing
- Overly formal constructions
- Missing commas (you will pause where the comma should be)

### Editing Ethics

- Preserve the author's voice while aligning it to the brand. Edit, do not ghost-write.
- Mark any substantive changes (altered meaning, added claims) clearly in the edit log.
- If a factual claim seems wrong, flag it — do not silently correct it unless you are certain.
- Never cut content that changes the author's argument without flagging it.

### Efficiency Tips

- Do not start line-editing a piece that needs structural rework. Fix structure first.
- Use the style guide as a lookup, not a memory exercise. Check it every session.
- If a piece is mostly clean, focus the editor's note on what is working and why — this builds the author's skill over time.
