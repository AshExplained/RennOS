**Team B workflow (Scrum with Design Phase):**
1. **Product Spec:** @product-designer (Dept 12) writes product spec
2. **Design Phase:** @ux-designer (Dept 14) generates UI screens in Stitch → @visual-designer (Dept 14) reviews for brand consistency → the CEO agent presents designs to the user (Sponsor) for **design freeze approval**
3. **Sprint Planning:** After the user approves designs → @scrum-master plans sprint from frozen designs + spec, creates Asana tasks
4. **Execution:** @tech-lead designs architecture → @full-stack-developer pulls Stitch HTML/CSS + implements → @qa-engineer tests → @tech-lead reviews PRs
5. **Deployment:** @devops-engineer deploys to staging → the user reviews → production
6. **Review:** @scrum-master runs sprint review + retro

**Roles:** The user = Sponsor (vision, design freeze, go/no-go). The CEO agent = Product Owner (backlog priorities, story acceptance). Sprint cycle: 2 weeks.
**Tools:** Team B uses Asana MCP for sprint management, GitHub CLI (`gh`) for code, Google Stitch MCP for design (via Dept 14 agents).
**Design freeze:** No coding starts until the user approves the designs. This is a hard gate.
